CLI からの実行

ブラウザにURLを入力してアプリケーションとして呼び出すコントローラと同様にコマンドラインのインターフェイス （CLI）を介してロードすることもできます。

  • CLI とは？
  • 何故コマンドラインから実行するの？
  • トライしてみよう:  Hello World!

CLI とは？

CLIはDOSのようにコンピューターとテキストベースの対話式でアプリケーションを操作する方法です。

何故コマンドラインから実行するの？

コマンドラインからCodeIgniterのを実行するための多くの理由がありますが、必ずしも明確な理由ではありません。

  • wget、またはcurlを使用しなくてもcronジョブを実行することができます。
  • IS_CLIがないかどうかチェックすることで、cronジョブにURLでアクセスできなくすることができます。
  • 対話式に"タスク"を作成することで、キャッシュフォルダや実行バックアップなどアクセス許可の設定を行うことができます。
  • 他の言語によるアプリケーションと統合します。 たとえば、ランダムC++のスクリプトを1つのコマンドとして呼び出したり、モデル内のコードを実行することができます！

通常では次のURLにアクセスします。:

example.com/index.php/tools/message/to

CLIでアクセスするにはMac/Lunixのターミナル、または、Windowsでコマンドプロンプトを起動してCodeIgniterプロジェクトのカレントディレクトリに移動します。

$ cd /path/to/project;
$ php index.php tools message

Hello World!と表示されるはずです。

$ php index.php tools message "John Smith"

URLの後ろに入力することでパラメータとして引数を渡すことができます。上の例では"John Smith"が引数として渡され表示されます。Hello John Smith!.

これだけ！

それは、一言で言えばすべてのコマンドライン上のコントローラの使い方を示しています。そのため、ルーティングと_remapは正常に動作し、普通のコントローラとして扱えることを覚えておいて損は無いでしょう。
